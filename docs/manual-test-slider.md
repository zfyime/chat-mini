# Zag Slider 升级后手动测试指南

本文档用于指导手动验证 Zag.js slider 从 0.16.0 升级到 1.42.0 后的功能正确性。

## 升级背景

- **升级时间**：2026-07-06
- **版本跨度**：`@zag-js/slider` 0.16.0 → 1.42.0（跨大版本）
- **主要变化**：API 从属性改为函数调用，value 从单值改为数组，回调参数结构变化
- **影响组件**：`src/components/Slider.tsx`（temperature 参数调节滑块）

## 测试步骤

### 1. 启动开发服务器

```bash
pnpm dev
```

访问 `http://localhost:4321`

### 2. 定位 Temperature 滑块

- 在聊天界面底部，点击**设置图标**（通常在输入框附近）
- 找到 **Temperature** 滑块控件
- 默认值应为 `0.6`（在 `src/config/constants.ts` 中定义）

### 3. 基础交互测试

#### 3.1 拖动测试
- **操作**：用鼠标拖动滑块的圆形手柄（thumb）
- **预期**：
  - 滑块跟随鼠标平滑移动
  - 滑块右侧的数值实时更新（保留两位小数）
  - 数值范围：`0.00` ~ `2.00`

#### 3.2 点击轨道测试
- **操作**：点击滑块轨道（track）的任意位置
- **预期**：
  - 滑块手柄立即跳转到点击位置
  - 数值同步更新

#### 3.3 边界值测试
- **操作**：将滑块拖到最左侧和最右侧
- **预期**：
  - 最小值：`0.00`
  - 最大值：`2.00`
  - 手柄不会超出轨道范围

#### 3.4 精度测试
- **操作**：缓慢拖动滑块
- **预期**：
  - 数值以 `0.01` 为步长变化（在 `Slider.tsx` 中定义为 `step: 0.01`）
  - 显示值始终保持两位小数格式

### 4. 视觉样式测试

#### 4.1 默认状态
- 滑块轨道为浅灰色背景
- 已选择范围（左侧填充部分）为深灰色
- 滑块手柄为圆形，带边框

#### 4.2 悬停状态
- **操作**：鼠标悬停在滑块手柄或轨道上
- **预期**：
  - 手柄颜色变浅
  - 填充范围颜色变化
  - 光标变为 `pointer`

#### 4.3 暗色模式
- **操作**：切换到暗色模式（如果项目支持）
- **预期**：
  - 滑块配色适配暗色主题
  - 对比度保持可读

### 5. 功能集成测试

#### 5.1 值持久化测试
- **操作**：
  1. 调整 temperature 到非默认值（如 `1.20`）
  2. 发送一条消息
  3. 刷新页面
- **预期**：
  - 重新打开设置，temperature 值保持为 `1.20`
  - 说明值正确保存到 localStorage/IndexedDB

#### 5.2 API 调用验证
- **操作**：
  1. 打开浏览器开发者工具 → Network 标签
  2. 调整 temperature 到 `0.90`
  3. 发送一条对话请求
  4. 查看 `/api/generate` 请求的 payload
- **预期**：
  - 请求体中包含 `"temperature": 0.9` 或 `"temperature": 0.90`
  - 值与滑块显示一致

### 6. 无障碍测试

#### 6.1 键盘导航
- **操作**：
  - 按 `Tab` 键聚焦到滑块
  - 使用 `←` `→` 方向键调整值
  - 使用 `Home` / `End` 键跳到最小/最大值（如果支持）
- **预期**：
  - 键盘操作响应灵敏
  - 数值按 `step` 步长变化

#### 6.2 屏幕阅读器
- **操作**：使用屏幕阅读器（如 macOS VoiceOver）聚焦滑块
- **预期**：
  - 读出 "Temperature" 标签
  - 读出当前值和范围
  - 操作提示清晰

### 7. 回归风险检查

#### 7.1 隐藏表单输入
- **操作**：在浏览器开发者工具 → Elements 面板检查滑块 DOM
- **预期**：
  - 存在一个 `<input type="hidden">` 元素
  - 其 `value` 属性与滑块显示值一致
  - 有 `data-scope="slider"` 和 `data-part="hiddenInput"` 属性

#### 7.2 Solid.js 响应式
- **操作**：
  1. 打开浏览器控制台
  2. 快速连续拖动滑块多次
  3. 观察数值更新是否有延迟或卡顿
- **预期**：
  - 数值更新流畅无延迟
  - 无控制台错误或警告

## 已知限制

- 当前升级仅通过构建验证和 lint，**未进行自动化 E2E 测试**
- 项目无 Playwright/Puppeteer 等浏览器自动化工具
- 需依赖人工手动测试确认运行时行为

## 如何报告问题

如果测试中发现问题，请记录：

1. **复现步骤**：详细描述操作顺序
2. **预期行为**：应该发生什么
3. **实际行为**：实际发生了什么
4. **截图/录屏**：如果可能，提供视觉证据
5. **浏览器信息**：浏览器类型和版本
6. **控制台日志**：是否有 JavaScript 错误

## 回滚方案

如果发现严重问题需要回滚：

```bash
# 恢复到 0.16.0 版本
pnpm add @zag-js/slider@0.16.0 @zag-js/solid@0.16.0

# 恢复组件代码（假设在 git 中）
git checkout HEAD~1 -- src/components/Slider.tsx src/styles/slider.css

# 重新构建
pnpm build
```
